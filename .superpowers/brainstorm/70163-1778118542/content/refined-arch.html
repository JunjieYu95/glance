Refined Architecture: Framework + Marketplace

Core skeleton separated from example trackers — professional, extensible

glance skill after clavhub install
glance/
├── SKILL.md   ← how to use the framework
├── install.sh
├── glance/   ← core framework skeleton
│  ├── cli.py
│  ├── core/   registry, storage, cron, auth
│  ├── dashboard/   HTML builder
│  └── skills/
│      └── scaffold_component/   meta-skill
└── examples/   ← marketplace examples
    ├── mood/   example tracker
    ├── reminder/   example tracker
    ├── mit/   example tracker
    └── diary_logger/   example tracker

How it works

clawhub install glanceDownloads framework + all examples
glance setupWizard: "Enable mood? [Y/n] Enable reminder? [Y/n]" — selected examples activated
glance scaffoldCreates USER trackers in ~/.glance/components/ — NOT inside the skill folder
glance dashboardBuilds from examples + user components

Design principles

  • Framework vs. examples — clear separation
  • User components live outside the skill — survive updates
  • Setup wizard: pick only what you want
  • Examples serve as reference for custom trackers
  • Community can publish tracker packs to ClawHub
