SKILL.md Design: Handbook + Dispatcher Pattern

Master SKILL.md routes intent to the right sub-skill

Master SKILL.md — the "handbook"
## What glance is
A personal tracker framework: mood, reminders, diary, MIT, dashboard.
Scaffold new trackers in one command.

## Routing user intent
| User says | Route to | Script to run |
|---|---|---|
| "log mood" / "how am I feeling" | examples/mood/SKILL.md | scripts/log.py |
| "remind me" / "add reminder" | examples/reminder/SKILL.md | scripts/log.py |
| "what's my MIT" | examples/mit/SKILL.md | scripts/today_brief.py |
| "log diary" / "track time" | examples/diary_logger/SKILL.md | scripts/log.py |
| "build dashboard" | → dashboard/build.py | |
| "create tracker for X" | → scaffold_component/SKILL.md | |

## For each route
1. First read [sub-skill SKILL.md] for full instructions
2. Then call the script with the arguments the user gave
3. Report back what happened

Flow

User intent → Master SKILL.md → Route to sub-skill → Call script

Example: "remind me to renew passport" → master sees "remind" → routes to examples/reminder/SKILL.md → agent reads it → runs scripts/log.py --add --title "renew passport"

Per-example SKILL.md (e.g. examples/mood/SKILL.md)

---
name: glance-mood
description: Hourly mood check-ins via chat
---

## Usage
glance mood log --raw "I feel great" --score 8
glance mood stats

## Scripts
scripts/log.py --raw "..." [--score 1-10]
scripts/stats.py   ← called by dashboard

Per-example SKILL.md files are optional companion skills — user can publish them on ClawHub as standalone skills too
