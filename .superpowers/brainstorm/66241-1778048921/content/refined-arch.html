Refined Architecture: Framework + Marketplace

Core skeleton separated from example trackers — professional, extensible

glance skill layout (ClawHub install)
glance/
├── SKILL.md   ← how to use the framework
├── install.sh
├── glance/   ← core framework (the skeleton)
│  ├── cli.py
│  ├── core/   registry, storage, cron, auth
│  ├── dashboard/   static HTML builder
│  └── skills/
│      └── scaffold_component/   meta-skill: create trackers
└── examples/   ← marketplace examples
    ├── mood/   example: mood tracker
    ├── reminder/   example: reminder tracker
    ├── mit/   example: MIT tracker
    └── diary_logger/   example: diary tracker

How it works

clawhub install glanceDownloads framework + all examples
glance setupWizard asks: "Enable mood? [Y/n] Enable reminder? [Y/n] ..." — only selected examples get activated (migrations run, cron registered)
glance scaffoldCreates NEW trackers under ~/.glance/components/ instead of inside the skill folder — user trackers stay separate from framework
glance dashboardBuilds dashboard from both selected examples + user-scaffolded components

Why this design

  • Clear separation: framework vs. content
  • User trackers live in ~/.glance/components/ — survive skill updates
  • Setup wizard lets users pick only what they want
  • Examples serve as reference for building custom trackers
  • Community can build and share tracker packs
