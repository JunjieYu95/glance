What's missing for install-as-a-skill

No OpenCode SKILL.md wrapper

Components have openclaw SKILL.md files, but there's no top-level OpenCode skill definition to wire everything together

No one-shot install flow

Users must clone the repo, run install.sh, configure openclaw.toml manually — no pipx/pip install + auto-configure

No predefined skeleton

No template/project generator for third-party users to start from scratch with their own dashboard + cron setup

Hardcoded to openclaw

Cron registration, notification paths, and SKILL.md formats are tightly coupled to openclaw's APIs
